# CPU VS GPU

## CPU(Central Processing Unit)

- 컴퓨터의 핵심 기능 구성 요소로 컴퓨터의 운영 체제와 애플리케이션을 실행하고 다양한 작업을 관리하는 전자 회로의 집합체
- 직렬 처리 방식을 사용하여 한 번에 하나씩 복잡하고 정교한 작업을 처리하는 데 최적화
- 구성
    - 연산장치
    - 제어장치
    - 레지스터
    - 캐시 메모리
- 중요성
    - 빠른 애플리케이션 실행: 지연 시간을 줄이고 사용자 경험을 향상
    - 효율적인 멀티태스킹: 성능 저하 없이 여러 애플리케이션 또는 스레드를 지원
    - 전력 최적화: 특히 모바일 및 임베디드 시스템에서 성능과 전력 소비 사이의 균형을 맞춤
    - 확장성: 저전력 엣지 디바이스부터 고성능 컴퓨팅 인프라에 이르기까지 다양한 시스템에 전력을 공급
- 주요 역할 및 특징
    - 파이프라이닝: 명령어 사이클을 중첩되는 단계로 나누어 처리량을 높임
    - 순서가 뒤바뀐 실행: 데이터 종속성 처리 중 유휴 시간을 줄이기 위해 명령어 순서를 재정렬
    - 분기 예측: 실행 지연을 최소화하기 위해 명령어 경로를 예측
        
        ⇒분기문이 많은 복잡한 알고리즘을 처리할 때 장점
        
    - 동시 멀티스레딩: 단일 코어가 여러 명령어 스트림을 동시에 관리할 수 있도록 함
    

## GPU(Graphics Processing Unit)

- 원래 그래픽 렌더링 속도를 높이기 위해 설계된 특수 프로세서
- 수천 개의 작은 코어를 통해 단순하고 반복적인 계산을 동시에 대량으로 처리(병렬 처리)하는 데 특화
- 여러 데이터를 동시에 처리할 수 있으므로 머신 러닝, 비디오 편집 및 게임 애플리케이션에 유용 ⇒최근에는 크리에이티브 제작 및 인공지능(AI) 분야에서도 점점 더 널리 사용
- 중요성
    - 단일 코어의 성능을 높이는 데에 대한 한계로 하드웨어를 특정 목적에 맞게 설계하는 가속기 시대가 도래함 → 그 중심에 GPU가 있음
    - 데이터 병렬성의 극대화
- 주요 역할 및 특징
    - 병렬 처리 : 단위 시간당 처리량(Throughput)을 극대화
    - 단순한 제어 로직과 큰 대역폭: 복잡한 분기 예측이나 거대한 캐시를 줄이는 대신, 연산 장치(ALU)의 밀도를 극대화

## CPU VS GPU

| **CPU (Central Processing Unit)** | **비교 항목** | **GPU (Graphics Processing Unit)** |
| --- | --- | --- |
| **Latency-Oriented** (지연 시간 최소화) | **설계 철학** | **Throughput-Oriented** (처리량 극대화) |
| 소수의 강력한 코어 (주로 4 ~ 64개) | **코어 수 / 구조** | 수천~수만 개의 단순한 코어 |
| 제어 로직(Control) 및 캐시(Cache) 비중이 큼 | **ALU vs Control/Cache** | 대부분의 다이 면적을 **ALU(연산 장치)**에 투자 |
| SISD, MIMD (독립적인 명령어 흐름) | **실행 모델** | SIMD, SIMT (동일 명령어 + 다중 데이터) |
| 분기 예측(Branch Prediction)으로 매우 뛰어남 | **분기 처리 능력** | 분기 분산(Branch Divergence) 발생 시 성능 저하 심함 |
| 낮은 대역폭, 낮은 지연 시간 (System RAM) | **메모리 특징** | 높은 대역폭(High Bandwidth), 높은 지연 시간 (VRAM) |

## 결론 및 요약

- **CPU**는 운영체제(OS) 관리, 파일 시스템 처리, 복잡한 비즈니스 로직 등 조건 분기가 많고 순차적인 제어가 필요한 'Task Parallelism'에 적합합니다.
- **GPU**는 행렬 곱셈, 딥러닝 학습/추론, 그래픽 렌더링 등 독립적인 대량의 데이터에 동일한 연산을 들이붓는 'Data Parallelism'에 압도적인 성능을 발휘합니다.
