# Compute Shader

- GPU에서 병렬 계산을 수행하는 프로그램
- 화면에 그림을 그리는 렌더링용 셰이더가 아니라 일반 연산을 빠르게 처리하기 위한 GPGPU 기술

## 1. Compute Shader의 작동 원리

- 대량의 연산을 수많은 쓰레드(Thread)로 쪼개어 동시에 처리
    - 스레드 (Thread): 데이터를 처리하는 최소 작업 단위
    - 스레드 그룹 (Thread Group): 스레드들이 모인 블록. 그룹 내의 스레드들은 고속 공유 메모리(Shared Memory)를 통해 서로 데이터를 주고 받기 가능.
    - 디스패치 (Dispatch): CPU가 GPU에게 "이 만큼의 스레드 그룹을 실행해라" 하고 명령을 내리는 단위

## 2. 주요 기능

- 대규모 병렬 연산 처리 : 많은 데이터를 한번에 나눠서 계산 가능
- 그래픽 파이프라인과 분리된 실행 : 버텍스, 픽셀 셰이더처럼 화면 렌더링 단계에 묶이지 않고 독립적으로 실행
- GPU 자원 활용 : GPU의 높은 병렬 처리 성능을 이용해 CPU보다 유리한 작업을 빠르게 처리
- 메모리 공유와 동기화 지원 : 스레드 그룹 단위로 작업하면서 공유 메모리와 동기화를 활용 가능

## 3. 핵심 특징

- 스레드 그룹 단위로 동작 : 여러 스레드가 같은 코드를 병렬로 실행
- 다양한 API 지원 : DirectX (HLSL), OpenGL/Vulkan (GLSL), Unity/Unreal 등 대부분의 현대 게임 엔진과 그래픽 API에서 표준으로 지원
- 입력과 출력을 직접 다룸 : 버퍼나 텍스처를 읽고 쓰는 식으로 데이터 처리가 가능
- 다양한 분야에 활용 : 물리 시뮬레이션, 입자 효과, 이미지 처리, AI연산, 과학 계산 등에 활용
- 디버깅과 메모리 관리가 까다로움 : GPU 병렬 처리 특성상 구조는 강력하지만 관리가 복잡.

## 4. 주로 어디에 쓰이나요?

- 물리 시뮬레이션 : 수만 개의 파티클(물방울, 연기, 먼지), 천(Cloth) 시뮬레이션, 군중 AI 계산.
- 포스트 프로세싱 (후처리 효과) : 화면 전체의 픽셀을 분석하여 블러(Blur), 블룸(Bloom), 톤매핑 등
- 게임 리소스 베이킹 및 컬링 : 화면에 보이지 않는 오브젝트를 빠르게 솎아내는 GPU 컬링(Occlusion Culling) 작업.
- 암호학 및 데이터 압축 : 그래픽과 상관없는 순수 수학적 연산이나 알고리즘 처리.

### 요약

Compute Shader는 화면 렌더링용 파이프라인에서 벗어나, GPU를 초고속 병렬 계산기로 쓸 수 있게 해주는 도구. 유니티(Unity)나 언리얼 엔진 같은 게임 개발은 물론이고 그래픽스 프로그래밍 전반에서 최적화를 위해 필수적으로 사용됨.
