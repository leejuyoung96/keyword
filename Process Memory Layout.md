## Code(Text) 영역

- 실행할 프로그램의 코드가 저장됨
- CPU는 이 영역에서 명령어를 하나씩 가져와 처리
- 프로세스가 종료될 때까지 유지
- 명령어를 변경하지 못하도록 읽기 전용(Read-Only) 인 경우가 많음.
- Heap과 Stack에 의해 메모리 공간이 덮어씌워지지 않도록 일반적으로 힙 또는 스택 메모리 공간 아래에 위치

## Data 영역

- 초기화된 전역변수와 Static 정적변수가 저장됨
- 프로그램 시작될 때 할당되어 프로그램 종료 시 소멸됨

## BSS(Block Stated by Symbol) 영역

- 초기화 되지 않은 전역 변수가 저장됨
- BSS 영역의 데이터는 프로그램 실행과 함께 커널(스타트업 코드)에 의해 0으로 초기화 됨

```csharp
/* Data 영역*/
static int i = 10; //초기값이 존재하는 정적 변수 i
int i = 0 ; //초기값이 존재하는 전역 변수 i

/* BSS 영역 */
static int i; //초기값이 없는 정적 변수 i
int i; //초기값이 없는 전역 변수 i
```

#### Data 영역과  BSS 영역이 구분되는 이유

- 메모리 공간을 효율적으로 사용하기 위함
- Data 영역: 초기값이 존재하는 데이터는 ROM(비휘발성) 메모리에 저장하되, 변수로써 사용하기 위해 프로그램 실행과 함께 RAM 메모리로 불러와 사용하는 것이 효율적
- BSS 영역 : 초기값을 가지지 않아 0으로 초기화 될 변수들이기에 ROM에 저장하지 않아도 됨. 프로그램을 수행하면서 0으로 초기화하여 RAM 메모리로 불러와 사용

## Stack 영역

- 지역변수, 매개변수, 리턴값 등 잠시 사용되었다 사라지는 데이터를 저장하는 영역
- 함수 호출 시 할당되어 Stack Frame이 생성되고, 함수 반환 시 소멸됨
- 프로그램 로드 시(컴파일 타임)에 크기가 결정됨
- 일반적으로 메모리 상위 주소에 위치함
- 스택 포인터(SP, Stack Pointer) 레지스터는 스택의 가장 상단의 위치를 가리키는 값을 가지고 있으며 스택이 push될 때마다 값이 조정됨

## Heap 영역

- 동적 데이터 영역
- 메모리 주소 값에 의해서만 참조되고 사용되는 영역(C의 malloc, JAVA/C#의 new)
- 프로그램 동작 시(런타임)에 크기가 결정됨
- BSS 영역의 끝 주소에서 시작
- 스택 포인터와 힙 포인터가 가리키는 주소가 같아지면 사용 가능한 메모리 공간이 모두 소진되었다는 의미
    - Heap Overflow : Heap 포인터가 Stack 영역을 침범함
    - Stack Overflow : Stack 포인터가 Heap 영역을 침범함
