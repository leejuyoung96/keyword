# 프로세스

- 실행중인 프로그램
- 운영체제에서 프로그램을 실행하는 작업 단위
- 프로세스는 프로세스를 실행하기 위한 작업의 최소 단위인 태스크(Task)를 완료하기 위해서 자원(resources)가 필요함
    - CPU의 점유, 메모리, 파일, 입/출력 장치
- 코드가 아무리 완벽해도 프로세스라는 실행 환경을 이해하지 못하면 대규모 서비스나 효율적인 소프트웨어를 만들 수 없음

⇒간단하게 메모리 위에 적재된 프로그램

## 프로세스의 구조

- **Code(Text) 영역** : 프로그래머가 작성한 소스 코드가 기계어 형태로 저장되는 영역. (Read-Only)
- **Data 영역** : 전역 변수(Global), 정적 변수(Static) 등 프로그램 시작과 동시에 할당되어 종료 시까지 유지되는 데이터 저장 영역.
- **Heap 영역** : 프로그래머가 동적으로 메모리를 할당하는 영역 (예: C의 malloc(), C#/Java/Python의 new 나 객체 생성). 런타임에 크기가 결정됨.
- **Stack 영역** : 함수 호출 시 생성되는 지역 변수, 매개 변수, 리턴 값 등이 저장되는 영역. 함수가 종료되면 자동으로 컴파일러에 의해 반환

## 프로세스의 상태

- 생성 (New) : 프로세스가 이제 막 메모리에 올라온 상태
- 준비 (Ready) : CPU를 할당받기 위해 대기하는 상태
- 실행 (Running) : CPU를 잡고 코드를 실행 중인 상태
- 대기 (Waiting/Blocked) : I/O 작업(파일 읽기, 네트워크 요청 등) 완료를 기다리며 잠시 멈춘 상태
- 종료 (Terminated) : 모든 실행이 끝나고 자원을 반납하는 상태

## 프로세스의 주요 기능

- **프로세스 제어 블록 (PCB - Process Control Block)** : 운영체제에서 각각의 프로세스는 PCB로 표현됨. PCB는 하나의 프로세스에 연관된 정보들을 포함함. 프로세스 ID(PID), CPU 레지스터 상태, 메모리 관리 정보 등이 담겨 있음.
- **문맥 교환 (Context Switching)** : CPU는 한 번에 하나의 프로세스만 실행 가능함. 빠르게 여러 프로세스를 번갈아 실행하기 위해, 기존 프로세스의 상태(PCB)를 저장하고 새 프로세스의 상태를 불러오는 작업. (단, 잦은 문맥 교환은 성능 저하(오버헤드)를 유발하므로 주의 필요)
- **프로세스 간 통신 (IPC - Inter-Process Communication)** : 프로세스는 기본적으로 서로의 메모리에 접근 불가. 따라서 프로세스끼리 데이터를 주고받기 위해 파이프(Pipe), 소켓(Socket), 공유 메모리(Shared Memory) 등의 IPC 기술을 활용함.

## 프로세스의 핵심 특징

- **독립성** : 각 프로세스는 자신만의 가상 메모리 공간을 가짐. 따라서 프로세스 A에서 에러가 발생해도 프로세스 B는 아무런 타격 없이 정상 작동함. (ex: 웹 브라우저에서 탭 하나가 멈춰도 다른 탭은 살아있는 이유)
- **동적 상태 전환** : 프로세스는 실행되면서 끊임없이 상태가 변함.
- **자원 소모가 큼 (Heavy-weight)** : 프로세스를 새로 생성하거나(Fork) 문맥 교환을 할 때, 메모리 공간을 통째로 세팅해야 하므로 스레드에 비해 비용(시간, 메모리)이 많이 들게 됨.

## 프로세스와 쓰레드의 구분

- 프로세스(Process)
    - 한개 이상의 쓰레드를 포함하는 실행중인 프로그램
    - 각 프로세스 간에는 자원이 독립적으로 할당됨
- 쓰레드(Thread)
    - 프로세스 안에 또 다른 작은 프로세스 → CPU가 프로세스를 실행시키기 위한 최소 작업 단위
    - 한 프로세스 안에 포함된 쓰레드들은 프로세스의 자원을 공유
